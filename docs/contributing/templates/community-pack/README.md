# Example community pack template

Copy this directory to `community/<pack_id>/` when preparing a new submission.

Replace placeholder names, paths, event-family details, and example validation text with source-specific content.

The purpose of this template is to show the minimum reviewable artifact shape:

- manifest metadata in `pack.json`
- SC4S parser/filter/context files as `.conf`/`.csv`
- Splunk knowledge objects as `.conf`
- fixtures under `test-events/`
- provenance and validation evidence under `docs/`

Do not submit the template itself as a real pack.
